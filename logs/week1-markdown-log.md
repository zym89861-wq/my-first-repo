# 📝 Week 1 协作日志：攻克 Markdown 与工作区搭建

- **时间**：2026年5月
- **参与者**：我 & Gemini Learning Agent
- **协作任务**：理清代码仓库架构逻辑，并深度理解 Markdown 与传统 Word/PDF 的应用场景区别。

### ⚡ 遇到的困惑
作为零基础跨界学习者，起初面对 `notes/`, `prompts/`, `demos/` 等一众英文文件夹感到迷茫，不理解在 AI 时代建立这种特定结构的必要性，同时也对 `.md` 文件的创建和排版原理缺乏概念。

### 🤖 Agent 如何协助我解决
1. **架构拆解**：Agent 将复杂的 GitHub 结构比作“高定服装工作室”，用极其通俗的语言梳理了各文件夹对沉淀 Prompt 和对接未来 AI 编程工具（如 Claude Code）的战略资产价值。
2. **实操避坑**：在创建并列目录时出现路径嵌套偏离，Agent 及时通过截图指出，指导我通过“回到根目录”纠正了路径。
3. **概念穿透**：用“毛坯房、精装房、样板房”的硬核比喻，直观讲透了 `.md` 文件在 AI 读取效率和跨平台防乱码上的本质优势。

### 💡 核心收割与复盘
AI 时代不需要死记硬背。通过本次协作，我不仅搭建好了专属的学习工作区，更掌握了利用 `#` 和 `*` 进行高效率、克制化专业排版的技能。后续将保持这种“场景导向、直接切入高频应用”的节奏，持续把 Web3 业务思考和 Agent 配置沉淀于此。
